Factorisation attacks

Given an RSA modulus n = p·q…

10.1 / Fermat

I O(|p−q|) Fermat's method

n = a² − b² = (a−b)(a+b),  a = ⌈√n⌉, ⌈√n⌉+1, …  until a²−n is a square

Write n as a difference of two squares…

Implementations:

What resists it
→ RSA → Pollard's rho → QR
10.2 / Pollard ρ (factoring)

II O(n^¼) Pollard's rho (factoring)

xᵢ₊₁ = xᵢ² + c mod n,  d = gcd(|x_i − x_j|, n),  1 < d < n ⇒ factor

The factoring twin of the discrete-log rho…

Same engine as §9.2 — see above.

Implementations:

What resists it
→ rho for DL → Pollard's p−1 → RSA
10.3 / Pollard p−1

II O(B log B) Pollard's p−1

M = lcm(1..B),  a^M mod n,  gcd(a^M − 1, n) reveals p  if p−1 is B-smooth

A special-purpose attack that wins when p − 1 is B-smooth…

Implementations:

What resists it
→ Pohlig–Hellman → Pollard's rho → RSA
10.4 / quadratic sieve

III L[1/2] Quadratic sieve (QS)

find x² ≡ y² (mod n),  x ≢ ±y  ⇒  gcd(x−y, n) is a factor

The fastest factoring method up to ~100 digits…

What resists it
→ GNFS → Tonelli–Shanks → index calculus
