Section 3

Block-cipher modes

The modes that turn a block cipher into a usable tool.

03 / XTS-AES

II O(n) XTS-AES

Cᵢ = E_K1(Pᵢ ⊕ Tᵢ) ⊕ Tᵢ, Tᵢ = E_K2(sector) · αⁱ in GF(2¹²⁸)

Security status
ECB/CBC/CTR GCM
03 / GCM-SIV

II O(n) GCM-SIV / SIV

SIV = PRF_K1(N, A, P); C = SIV ‖ ( P ⊕ CTR_K2(SIV) )

Security status
ECB/CBC/CTR GCM
03 / OCB

II O(n) OCB

L = E_K(0); Δᵢ = offset(L, i); Cᵢ = E_K(Pᵢ ⊕ Δᵢ) ⊕ Δᵢ; tag = E_K(Σ Pᵢ ⊕ Δ*)

Security status
ECB/CBC/CTR GCM
03 / EAX

II O(n) EAX

N = OMAC⁰(nonce); C = P ⊕ CTR_K(N); tag = OMAC⁰(nonce) ⊕ OMAC¹(header) ⊕ OMAC²(C)

Security status
ECB/CBC/CTR GCM
03 / AES-KW

I O(n) AES Key Wrap

A = A6A6…; for j=0..5, i=1..n: B = E_K(A ‖ Rᵢ), A = MSB(B) ⊕ t, Rᵢ = LSB(B), t = n·j+i

Security status
ECB/CBC/CTR GCM HMAC
03 / FPE

III O(n) FF1 / FF3

Feistel over base-10: Aᵢ₊₁ = B, Bᵢ₊₁ = ( num(A) + F_K(i, B, tweak) ) mod 10ᵐ

Security status
ECB/CBC/CTR GCM AES
03 / EtM vs MtE

II — EtM vs MtE

EtM: (C, MAC(C)) · MtE: E(P ‖ MAC(P)) · E&M: (E(P), MAC(P))

order authenticates failure mode verdict
Encrypt-then-MAC the ciphertext C none generic — bad C rejected before decrypt ✓ safe
MAC-then-Encrypt the plaintext P (inside C) padding oracles: POODLE, Lucky-13 ⚠ fragile
Encrypt-and-MAC P and C separately MAC(P) can leak; no C integrity ✗ avoid
Security status
ECB/CBC/CTR GCM Padding oracles HMAC
